# Git Commit Diary Desktop Float PRD

## 1. 产品概览

| 项目 | 内容 |
|---|---|
| 产品名称 | Git Commit Diary Desktop Float |
| 产品定位 | 面向开发者的 Windows 桌面悬浮 Git 工作流日记助手 |
| 核心价值 | 把 Git 提交、分支状态、今日开发进展转成可快速查看和复用的开发日记 |
| 首版形态 | 桌面悬浮小窗口 + 托盘常驻 |
| 首版策略 | 状态窗 + 快捷操作 |
| 生成方式 | 本地规则底稿 + AI 增强 |
| 目标平台 | Windows 单平台 |

## 2. 用户场景

| 场景 | 目标 |
|---|---|
| 写代码时扫一眼 | 查看当前仓库、分支、今日提交数、最近提交、待提交变更 |
| 准备日报 | 一键生成今日开发日记 |
| 快速复用 | 一键复制日报文本 |
| 定位上下文 | 快速打开仓库目录 |
| 后台常驻 | 通过托盘显示、隐藏、退出、进入设置 |

## 3. MVP 功能

| 功能 | 说明 |
|---|---|
| 悬浮窗 | 可拖拽、置顶、折叠、展开、记住位置 |
| Git 状态扫描 | 读取仓库名、分支、今日提交、最近提交、待提交变更 |
| 本地日记底稿 | 按提交类型、文件范围、变更摘要生成 Markdown |
| AI 增强 | 对底稿做归纳、润色、日报化表达、待办提取 |
| 快捷复制 | 复制最近一次日报内容到剪贴板 |
| 托盘菜单 | 显示窗口、隐藏窗口、选择仓库、设置、退出 |
| 本地存储 | 保存仓库路径、窗口位置、偏好配置、日记历史 |

## 4. 悬浮窗信息层级

| 层级 | 内容 |
|---|---|
| 顶部 | `CommitDiary · main` |
| 核心状态 | `今日 5 commits · 2 changes` |
| 最近提交 | `feat: add diary generator` |
| 状态提示 | 已生成、增强中、复制成功、扫描失败 |
| 快捷按钮 | 刷新、生成、复制、展开 |
| 展开视图 | 今日提交列表、日记预览、影响范围、待办事项 |

## 5. 交互行为

| 交互 | 规则 |
|---|---|
| 点击生成 | 先生成本地规则底稿，再触发 AI 增强 |
| 点击复制 | 复制最近一次已生成日报 |
| 点击展开 | 打开详情面板，展示完整 Markdown 预览 |
| 点击仓库名 | 打开仓库目录 |
| 托盘隐藏 | 窗口隐藏，后台继续监听当前仓库 |
| 扫描失败 | 展示失败原因和重试按钮 |

## 6. 日记生成策略

| 项目 | 设计 |
|---|---|
| 基础生成 | 本地规则读取 Git 提交、分支、文件变更、工作区状态，生成稳定底稿 |
| AI 增强 | AI 对底稿做归纳、标题化、日报化表达、风险与待办提取 |
| 离线可用 | 本地规则可独立生成基础日记 |
| 智能增强 | 配置 AI 后生成更自然、更适合日报和周报复用的内容 |
| 用户控制 | 用户可选择基础版、增强版，也可重新生成 |
| 输出格式 | Markdown 文本，便于复制到飞书、Notion、GitHub、日报系统 |
| 数据边界 | 首版读取本地 Git 元数据、commit message、文件路径、diff 摘要 |
| 隐私策略 | 默认在本地完成采集和模板整理，AI 调用前展示将发送的摘要级上下文 |

## 7. 生成结果结构

| 模块 | 内容 |
|---|---|
| 今日概览 | 今天主要完成了什么 |
| 提交摘要 | 按提交聚合关键改动 |
| 影响范围 | 涉及模块、文件类型、功能区域 |
| 待处理事项 | 从 WIP、TODO、未提交变更中提取 |
| 复制版日报 | 一段可直接粘贴的日报文本 |

## 8. 生成流程

| 步骤 | 说明 |
|---|---|
| 1. 扫描仓库 | 获取分支、今日提交、工作区状态 |
| 2. 规则归类 | 按 feat/fix/refactor/docs/test/chore 归档 |
| 3. 生成底稿 | 输出结构化 Markdown 开发日记 |
| 4. AI 增强 | 生成更自然的日报表达 |
| 5. 缓存结果 | 保存今日日记、生成时间、复制内容 |
| 6. 快捷复用 | 用户一键复制或展开查看 |

## 9. 模块边界

| 模块 | 职责 |
|---|---|
| 桌面壳层 | Windows 悬浮窗、托盘、剪贴板、打开目录 |
| Git 采集模块 | 获取仓库、分支、提交、变更摘要 |
| 状态计算模块 | 生成悬浮窗实时状态 |
| 日记生成模块 | 生成本地 Markdown 底稿 |
| AI 增强模块 | 生成增强版日报文本 |
| 本地存储模块 | 保存配置、历史、窗口位置 |
| UI 渲染模块 | 渲染小窗、展开详情、设置入口 |
| 错误处理模块 | 处理仓库、Git、AI、权限、剪贴板异常 |

## 10. 核心数据流

| 步骤 | 说明 |
|---|---|
| 1. 选择仓库 | 用户选择一个本地 Git 仓库路径 |
| 2. 定时扫描 | 后台按固定间隔读取 Git 状态 |
| 3. 状态刷新 | 悬浮窗更新分支、提交数、最近提交、变更数 |
| 4. 生成底稿 | 用户点击生成后，本地规则生成日记底稿 |
| 5. AI 增强 | 已配置 AI 时生成增强版日记 |
| 6. 缓存结果 | 保存今日日记、生成时间、复制内容 |
| 7. 快捷复用 | 用户一键复制日报或展开查看详情 |

## 11. 异常处理

| 异常场景 | 处理 |
|---|---|
| 仓库路径异常 | 提示选择有效 Git 仓库，保留最近一次有效状态 |
| Git 扫描失败 | 展示失败原因、重试按钮、最近成功扫描时间 |
| 今日零提交 | 展示空状态，并支持生成基础日记 |
| 存在待提交变更 | 在悬浮窗展示数量，并进入日记待办 |
| AI 增强失败 | 保留本地规则底稿，展示增强失败原因 |
| 复制失败 | 提示剪贴板写入失败，展开后可选择文本 |
| 权限异常 | 提示仓库访问或系统权限问题，引导重新选择路径 |
| 网络异常 | 影响 AI 增强，本地规则生成保持可用 |
